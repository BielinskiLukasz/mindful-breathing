<task type="auto">
  <name>Task 2: Uniform hover states for all interactive controls (D-08, UX-01)</name>
  <files>index.html</files>
  <action>
    One CSS addition. Add a new section at the end of the `<style>` block, immediately before `</style>`. Use the existing section-divider convention:

      `/* ====== HOVER STATES (pointer: fine) ====== */`
      `@media (hover: hover) and (pointer: fine) {`
      `  button {`
      `    transition: filter 120ms ease, transform 120ms ease;`
      `  }`
      `  button:hover {`
      `    filter: brightness(1.08);`
      `    transform: scale(1.02);`
      `  }`
      `}`

    The `button` selector covers ALL interactive controls: Start, Reset (resetBtn), preset buttons (presetBtn), icon toggles (iconToggle — which are buttons), infoBtn, settingsBtn, soundToggle, vibeToggle, themeToggleBtn, fullscreenBtn (now in cornerControls), clearHistBtn, histPrevBtn, histNextBtn, exportJsonBtn, exportCsvBtn, importBtn, settingsCloseBtn, infoCloseBtn, doneRestartBtn, countdownSkipBtn. No exceptions — uniform per D-08.

    The `transition` is placed on the BASE state (not inside :hover) so that both the hover-enter and hover-leave transitions are smooth. The `@media (hover: hover) and (pointer: fine)` guard ensures touch devices (phones, tablets) receive no hover effect.

    These properties ADD to existing `:hover` rules without conflict. Existing hover rules set background/border/color — this adds filter and transform on top. CSS specificity allows both to coexist cleanly.

    DO NOT add `will-change: transform` — the transform is minor (scale 1.02) and does not warrant layer promotion.
    DO NOT override or remove any existing `.presetBtn:hover`, `.infoBtn:hover`, `.fullscreenBtn:hover`, or `.doneRestartBtn:hover` rules — those color/background changes remain.
  </action>

    - Hover over fullscreen button: same effect
    - On mobile (Chrome DevTools mobile emulation): no scale/brightness change on tap
    - Transition on hover-leave is also smooth (button returns to normal at 120ms)
  </verify>
