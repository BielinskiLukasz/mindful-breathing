<task type="auto">
  <name>Task 1: Create Landscape Media Query with Flex Layout and Ring Scaling</name>
  <files>index.html</files>
  <action>
Add @media (orientation: landscape) rule block to the CSS section (approximately after existing media queries for max-width: 480px and max-width: 360px, around line 1200-1300).

Create a complete landscape media query with the following structure:

```css
@media (orientation: landscape) {
  body {
    display: flex;
    flex-direction: column;
    align-items: center;
    justify-content: center;
    min-height: 100vh;
  }

  .container {
    display: flex;
    flex-direction: row;
    width: 100%;
    max-width: 1400px;
    height: 100vh;
    padding: 0;
  }

  .ringSection {
    flex: 1;
    display: flex;
    align-items: center;
    justify-content: center;
    padding: 20px;
    min-width: 0;  /* Allow flex shrinking on small landscape phones */
  }

  .ringWrap {
    width: clamp(200px, 25vw, 400px);
    height: clamp(200px, 25vw, 400px);
  }

  .controlsSection {
    flex: 1;
    display: flex;
    flex-direction: column;
    justify-content: flex-start;
    padding: 20px;
    overflow-y: auto;
    border-left: 1px solid var(--accentSoft);
  }
}
```

**Rationale per each CSS rule:**

1. **body display: flex; flex-direction: column:** Centers the main container on the screen (landscape mode fills both width and height)

2. **.container display: flex; flex-direction: row:** Splits layout into left/right sections

3. **.container max-width: 1400px:** Caps layout on very wide screens (4K, 3840px+). Without this, ring and controls would stretch indefinitely.

4. **.container padding: 0:** Removes padding on landscape (width: 100% fills screen). Centering is handled by body flex + max-width.

5. **.ringSection flex: 1; min-width: 0:** Gives ring section 50% of available space. `min-width: 0` is critical—it allows flexbox to shrink the ring section on small landscape phones (480px wide × 360px tall) without overflow.

6. **.ringWrap width/height clamp(200px, 25vw, 400px):** Ring scales fluidly with viewport width. `25vw` is approximately 25% of landscape viewport width (which is the width of a 50% section). Min 200px ensures ring doesn't vanish on tiny screens; max 400px prevents oversized ring on ultra-wide displays.

7. **.controlsSection flex: 1; min-width: 0:** Gives controls/history 50% of available space. `min-width: 0` prevents overflow.

8. **.controlsSection flex-direction: column; overflow-y: auto:** Stacks controls and history vertically. `overflow-y: auto` allows history to scroll independently while controls remain visible at bottom (see Task 2).

9. **.controlsSection border-left:** Subtle visual divider between ring and controls (uses --accentSoft which adapts to theme). Optional but recommended for clarity.

**Implementation notes:**
- Do NOT modify .ringSection or .controlsSection in portrait media queries (@media max-width); landscape rules will override them
- Ensure .ringWrap exists as a child of .ringSection in HTML (it should; SVG ring is wrapped in this div)
- Verify no conflicting display or flex properties in parent selectors that might override landscape rules
- Keep portrait layout intact; this is additive only

Save the complete media query to CSS block.

<task type="auto">
  <name>Task 2: Position Controls Fixed at Bottom, History Scrollable Above</name>
  <files>index.html</files>
  <action>
Within the @media (orientation: landscape) rule, add rules to position controls at the bottom of the right section and make history scrollable above. This requires restructuring the flex layout within .controlsSection.

Identify the HTML structure for controls (buttons, presets, toggles) and history in the right section. Typically:
- `.controls` or button group (Start, Stop, Reset, presets, toggles)
- `.history` or `#historyList` (session history cards)

Within the landscape media query, add:

```css
  .history {
    flex: 1;
    overflow-y: auto;
    margin-bottom: 16px;
    padding-right: 8px;  /* Small right padding to prevent scrollbar overlap */
  }

  .controls {
    flex-shrink: 0;  /* Prevent controls from shrinking */
    display: flex;
    flex-wrap: wrap;
    gap: 8px;
    padding: 8px 0;
    border-top: 1px solid var(--accentSoft);
  }
```

**Explanation:**

1. **.history flex: 1; overflow-y: auto:** History takes remaining vertical space and scrolls independently. `flex: 1` makes it expand to fill available height (after controls size themselves).

2. **.history margin-bottom: 16px; padding-right:** Adds separation from controls and scrollbar spacing.

3. **.controls flex-shrink: 0:** Prevents controls from shrinking. They should always be fully visible (Start, Stop, Reset buttons, etc.).

4. **.controls flex-wrap: wrap:** Allows buttons to wrap to multiple lines if space is tight. Ensures all controls remain accessible.

5. **.controls border-top:** Visual separator between history and controls (uses --accentSoft for theme adaptation).

**Alternative if HTML structure is different:**
If the HTML uses different class names (e.g., `.controlPanel` instead of `.controls`), adjust selectors accordingly. The goal is:
- History scrolls in a flex: 1 container
- Controls remain fixed size (flex-shrink: 0) at bottom
- Visual separator between them

**Verify the layout:**
After implementing, the landscape layout should have:
- Ring on left (50%)
- History panel on right, scrollable, taking most vertical space
- Controls (buttons) at bottom of right section, always visible, not scrolling
- No horizontal scroll at any width
