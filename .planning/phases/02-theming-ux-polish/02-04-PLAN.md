<task type="auto">
  <name>Task 3: Test Responsive Typography (Plan 2 baseline)</name>
  <files></files>
  <action>
Test font size scaling across the full device range. Plan 2 uses clamp() to scale fonts fluidly; verify the scaling is smooth and readable.

**For each device size, visually inspect font sizes:**

| Element | iPhone 12 (390px) | iPad (768px) | Desktop (1280px) | 4K (2560px) |
|---------|-------------------|--------------|------------------|------------|
| Body text | ~12px | ~14px | ~16px | 16px (clamped) |
| Phase label | ~20px | ~24px | ~28px | 28px (clamped) |
| Countdown timer | ~48px | ~56px | ~64px | 64px (clamped) |
| Button text | ~13px | ~15px | ~17px | 17px (clamped) |
| History card | ~11px | ~12px | ~14px | 14px (clamped) |

**Test procedure:**

1. **Open app on iPhone 12 (390px) portrait**
   - Visually estimate countdown timer size — should be large but not enormous (~48px)
   - Phase label (Inhale/Exhale) should be smaller than timer but larger than body (~20px)
   - Body text (status, buttons) should be clearly readable (~13px+)
   - History card text should be compact but readable (~11px+)
   - No text should be too small to read comfortably

2. **Resize to iPad (768px) portrait**
   - All fonts should visibly increase (smooth scaling, no sudden jump at breakpoint)
   - Countdown timer larger (~56px)
   - All other text proportionally larger
   - No cramping; plenty of breathing room

3. **Resize to Desktop (1280px) landscape**
   - Fonts larger again (~28px phase, ~64px timer)
   - Text should be easily readable from typical viewing distance
   - Maintain visual hierarchy (timer > phase > body > history)

4. **Resize to 4K (2560px) landscape**
   - Fonts clamp at maximum (timer ~64px, phase ~28px, body ~16px)
   - Should not continue growing (verify clamp() is working)
   - Text should not be oversize for the screen

5. **Resize back to 320px (extra small) portrait**
   - Fonts clamp at minimum (timer ~40px, phase ~18px, body ~12px)
   - Verify no text is too small to read

6. **Slow resize (dragging DevTools width slider) from 320px to 2560px**
   - Watch fonts scale smoothly over full range
   - Should see continuous scaling, NO sudden jumps at 480px or 360px breakpoints
   - If jumps occur: Plan 2 may have leftover fixed font-size rules in media queries

**Document findings:**
- [ ] Font sizes readable on smallest device (320px)
- [ ] Font sizes don't exceed comfortable range on largest (4K)
- [ ] Scaling smooth (no breakpoint jumps when resizing)
- [ ] Visual hierarchy maintained (timer > label > body > history)
- [ ] All text elements scale (phase, timer, body, buttons, history)
- [ ] Line-height and spacing feel natural at all sizes

Common issues to watch for:
- Text jumps at specific breakpoints (old fixed font-size rules not fully replaced)
- Text too small on phone (clamp min value too low)
- Text too large on 4K (clamp max not capped properly)
- One element doesn't scale (selector not updated with clamp)
- Line-height too tight on phone or too loose on desktop (proportional scaling not applied)
