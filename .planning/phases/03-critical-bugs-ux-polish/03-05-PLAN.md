  <how-to-verify>
**Equipment needed:** 2+ Android devices (phone and/or tablet preferred; different manufacturers if available)

**Test Setup:**
1. Deploy app to staging or open GitHub Pages: https://bielinskilukasz.github.io/mindful-breathing/
2. Open in Chrome or Edge browser on Android device
3. Ensure vibration is enabled (not in silent/mute mode for first test)
4. Ensure sound is enabled (optional; vibration is primary focus)

**Test Scenarios:**

**Scenario 1: Phase Transition Vibration (All Phases)**
- [ ] Click Start button (countdown 3s with vibration pulses)
- [ ] Observe 3 vibration pulses during countdown (1 pulse per second: 3s, 2s, 1s) ✓ / ✗
- [ ] Session starts; observe Inhale phase begins
- [ ] At Inhale→Hold transition: feel single 50ms vibration pulse ✓ / ✗
- [ ] At Hold→Exhale transition: feel single 50ms vibration pulse ✓ / ✗
- [ ] At Exhale→Hold2 transition: feel single 50ms vibration pulse ✓ / ✗
- [ ] At Hold2→Inhale transition (next cycle): feel single 50ms vibration pulse ✓ / ✗
- [ ] Continue for 2+ cycles; confirm vibration consistent across all phase transitions

**Scenario 2: Silent Mode Behavior**
- [ ] With device in **Silent mode** (physical mute switch or OS settings)
- [ ] Start a session
- [ ] Attempt to feel vibration during phase transitions ✓ (vibrates in silent) / ✗ (muted in silent) / ⚠️ (inconsistent)
- [ ] Document behavior: Does device vibrate in silent mode or is haptics disabled?

**Scenario 3: Low Battery / Power Saver**
- [ ] Enable low-battery/power-saver mode (if device allows)
- [ ] Start session
- [ ] Verify vibration still fires or note if disabled by OS ✓ / ✗ / ⚠️

**Scenario 4: Session Stop & Resume**
- [ ] Start session; feel vibration on phase transitions ✓
- [ ] Click Stop at 3-minute mark
- [ ] Immediately click Start again
- [ ] Verify countdown vibrations fire again ✓
- [ ] Verify phase transition vibrations resume ✓

**Data Collection for Report:**

For each device tested, record:
- Device model (e.g., Samsung Galaxy S21, Google Pixel 6)
- Android version (e.g., Android 12, 13)
- Browser (Chrome or Edge)
- Test date
- Results for each scenario (✓ pass, ✗ fail, ⚠️ partial)
- Notes (e.g., "Vibration disabled in silent mode", "Pattern felt as single pulse", "Countdown pattern is 3 distinct pulses")

**Acceptance Criteria:**
- Phase transition vibration works on at least 2 Android devices
- Countdown vibration works (3 pulses during 3-second countdown)
- Pattern is consistent (50ms single pulse per phase transition)
- Chrome and Edge browsers both support Vibration API (no browser-specific failures)
- Silent mode behavior is documented (device mutes haptics or continues)
