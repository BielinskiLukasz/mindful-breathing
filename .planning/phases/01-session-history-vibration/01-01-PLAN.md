<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User ↔ Settings Panel | User provides numeric input for max-logs cap; trusted (own device) but validated (1–10,000 bounds) |
| App ↔ localStorage | Untrusted source: old data may lack preset field; handled gracefully (defaults to #9aa0a6 gray badge) |
| App ↔ Browser APIs | navigator.vibrate() and localStorage trusted; feature detection guards against undefined APIs |
| Session State ↔ SessionStorage | Page number stored in sessionStorage; low sensitivity (resets on page reload) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | localStorage history | Accept | No sensitive data (dates, durations, preset names are public). User-initiated writes only via normal session flow. No external input validation needed. |
| T-01-02 | Information Disclosure | Max-logs cap value | Accept | Cap value is user-configurable; no sensitive information. Stored in STORAGE_KEY alongside other settings. |
| T-01-03 | Denial of Service | Vibration spamming | Mitigate | Vibration calls come only from `advanceToNextPhase()` (phase transitions) or `showCountdown()` (countdown overlays), both rate-limited by phase durations (min 1 sec). No user input controls vibration frequency. Graceful degradation if API unsupported. |
| T-01-04 | Information Disclosure | Preset badge rendering | Accept | Preset names (Relax, Box, 4-7-8) are public UI labels. Colors from PRESETS config (public). No user input exposed. |
| T-01-05 | Tampering | Pagination page number | Accept | Page number stored in sessionStorage (not persistent, resets on reload). Low-value target; users can only navigate within their own history. No data loss if page number invalid (renders as page 1). |

**No new packages added** — all Phase 1 changes use vanilla JS and built-in browser APIs. No npm/pip/cargo install tasks required.
