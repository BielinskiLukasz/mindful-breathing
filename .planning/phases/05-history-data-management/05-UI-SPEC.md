## Visual Focus Hierarchy

**History Panel Actions (`#historyActions` button row):**

| Button | Hierarchy | Sizing | Border Highlight |
|--------|-----------|--------|-------------------|
| Export JSON | Primary | Standard preset button size (~44px min touch target) | Subtle accent border (inherits from `.presetBtn` default) |
| Export CSV | Secondary | Matches Export JSON (visual pairing) | Inherits from `.presetBtn` |
| Import | Secondary | Matches other buttons | Inherits from `.presetBtn` |

**Design Rationale:** Export JSON is listed first and serves as the primary download action (broadest use case â€” preserves full precision data). Export CSV is secondary (format choice). Import is secondary (less frequent operation). All three buttons use the same visual treatment (`.presetBtn` style) to keep the row compact and cohesive. No button is visually prominent over the others (no bold, no color distinction).

**Dialog Actions (`#clearConfirmDialog` buttons):**

| Button | Hierarchy | Styling |
|--------|-----------|---------|
| Delete all sessions | Destructive | Implementer defines (e.g., red background, white text, or inverted style) |
| Cancel | Primary action | Default button style (matches `.presetBtn`) |

**Design Rationale:** Cancel is the safe/primary action (returns to normal state). Delete is destructive and should be visually distinguished by the implementer using color or prominence to reduce accidental clicks.

### Accent Reservation

**10% accent color (`--accent`) is reserved for:**
- Primary button focus state (Start button when idle)
- Active preset badge highlight (`.presetBtn.active`)

- Phase label color during animation
- Button borders on hover/active states
- Ring/progress indicator stroke

**Not used for:** Neutral icons, secondary buttons (those use `--accentSoft`), destructive actions (no secondary color in this phase).

## Component Interaction & Behavior

### Export JSON Button

- **Trigger:** User clicks "Export JSON"
- **Action:** Serialize all entries from `localStorage["mb_history"]` to JSON, wrap in Blob
- **Download:** Create temporary `<a>` with `URL.createObjectURL(blob)`, set `.download` filename, programmatically click, revoke URL
- **Filename:** Date-stamped using `new Date().toISOString().slice(0, 10)` (YYYY-MM-DD)
- **File content:** JSON array of `{ date, durationMs, cycles, preset }` objects

### Export CSV Button

- **Trigger:** User clicks "Export CSV"
- **Action:** Serialize all entries to CSV with headers: `date, duration, cycles, preset`
- **Duration format:** Use existing `formatDuration(ms)` function to display as `M:SS` (e.g., "5:32")
- **Date column:** Full ISO timestamp (e.g., "2026-06-30T11:45:38.188Z")
- **Download:** Same Blob + `<a download>` pattern as JSON
- **File content:** CSV with quoted values where necessary, RFC 4180 compliant (no unquoted newlines)

### Import JSON Button

- **Trigger:** User clicks "Import"
- **Action:** Click hidden `<input type="file" accept=".json">` file picker
- **FileReader:** Read file as text, `JSON.parse()`, validate each entry
- **Schema validation:** Each entry must have `date` (string), `durationMs` (finite number), `cycles` (finite number); `preset` optional (default to "relax")
- **Merge logic:**
  - Read existing history from `localStorage["mb_history"]`
  - Skip any imported entries where `entry.date === existingEntry.date` (exact ISO string match)
  - Combine valid + imported entries
  - Re-sort by date descending (newest first: `b.date.localeCompare(a.date)`)
  - Enforce `maxHistoryEntries` cap after merge
  - Write back to localStorage atomically (read â†’ merge â†’ write once)
- **Feedback:** Show 5-second flash message on `#storageWarning`:
  - Success: `"Imported X sessions (Y duplicates skipped)"`
  - Partial: `"Imported X sessions (Y invalid entries skipped)"`
  - Failure: `"Import failed: invalid file format"`
- **Error handling:** Wrap in try/catch; catch `SyntaxError` (malformed JSON) and `QuotaExceededError` (storage full); both trigger failure message
- **Refresh:** Call `renderHistory()` to update the displayed list after successful import

### Clear History Confirmation Dialog

- **Trigger:** User clicks "Clear" button in history summary
- **Dialog opens:** Native `<dialog>` element with:
  - Heading: `"Delete all sessions?"`
  - Confirm button: `"Delete all sessions"` (destructive language)
  - Cancel button: `"Cancel"`
  - Focus trap: native `<dialog>` provides free focus trap
  - Dismissal: ESC key or click outside â†’ dialog closes, NO deletion
- **Confirm action:** Delete all entries from `localStorage["mb_history"]`, reset `historyPage` in `sessionStorage`, call `renderHistory()` to show empty state
- **Empty state text:** `"No sessions yet"` (reuse existing `.historyEmpty` element)
- **After deletion:** `<dialog>` closes, user sees empty history panel
