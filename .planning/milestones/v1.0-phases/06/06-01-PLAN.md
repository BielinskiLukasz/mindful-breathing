  <name>Task 2: Fix compact landscape body height to allow content scroll (LAYOUT-01, D-14, D-15)</name>
  <files>index.html</files>
  <read_first>
    - index.html lines 887–944 (full compact landscape media query block)
    - index.html lines 43–54 (base body rule — height: 100vh inherited into compact landscape)

    - .planning/phases/06/06-CONTEXT.md decisions D-14 and D-15
  </read_first>
  <action>
    The compact landscape media query is `@media (max-height: 500px) and (orientation: landscape)`.
    The media query CONDITION is correct and must NOT be changed (per D-14).

    The bug: the base body rule has `height: 100vh`, which is never overridden in the compact landscape
    body block. When the compact layout's content (ring + controls + history) is taller than the viewport
    (e.g., 600px content in a 430px viewport), the body is clamped to 100vh. The body's own
    `overflow-y: auto` can scroll, but the fixed 100vh height prevents the natural document scroll that
    mobile browsers handle more reliably (auto-hiding the URL bar, etc.).

    Inside `@media (max-height: 500px) and (orientation: landscape)`, find the `body { ... }` rule
    (lines 888–894). Add these two properties to that rule:

      height: auto;
      min-height: 100vh;

    `height: auto` removes the 100vh constraint — the body expands to fit the container.
    `min-height: 100vh` ensures the body is at least viewport height when content is short (preserves
    vertical centering behavior for short content).

    Per D-15: ring + controls must be visible immediately (they appear at the top of the container).
    The history div already has `max-height: 180px; overflow-y: auto` (existing, do not change),
    keeping it scrollable within its own region below the fold.

    Do NOT add or remove any other properties in this media query.
  </action>
