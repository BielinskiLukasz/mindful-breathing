## v0.5 Requirements

### Bug Fixes

- [x] **BUG-01**: Countdown timer orphan prevention — `stop()` must clear countdown timers when interrupted mid-countdown
- [x] **BUG-02**: Duration input validation — Phase duration inputs clamped to 1–300 seconds with shared constant
- [x] **BUG-03**: localStorage quota error handling — `QuotaExceededError` surfaced to user with visible warning

### Micro-Interactions

- [x] **MICRO-01**: Button press feedback — `:active` state provides tactile visual press-down cue
- [x] **MICRO-02**: Phase transition ring flash — SVG ring brightens briefly on each phase advance
- [x] **MICRO-03**: Gesture hint onboarding — First-visit hint shown once, dismissed on any interaction
- [x] **MICRO-04**: Duration input error flash — Red highlight flash on invalid/out-of-range input

### Testing & Validation

- [x] **TEST-01**: Android vibration validation — Validated on Samsung Galaxy A33 5G; non-functional on Android 16 Chrome. Closed as `known_limitation` — vibration UI hidden
- [x] **TEST-02**: Countdown audio pattern — 3 audible beeps (350 Hz) at 3s/2s/1s countdown marks
- [x] **TEST-03**: Cross-browser testing (Chrome/Edge) — Verified on Chrome Android and Edge desktop; audio, wake lock, and FPS behavior confirmed
