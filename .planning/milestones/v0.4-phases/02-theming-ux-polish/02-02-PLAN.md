  <files>index.html</files>
  <action>
Apply proportional scaling to line-height and letter-spacing using clamp() or proportional calculations (per RESEARCH.md recommendation).

**Line-height scaling (applies globally to body or specific text containers):**
Add or update body-level line-height to use clamp():
```css
body {
  line-height: clamp(1.4, 0.5vw + 1.2, 1.8);
}
```

If line-height is already defined on body, replace the fixed value (e.g., `line-height: 1.6`) with the clamp() formula above.

**Letter-spacing (optional, applies to headings/labels for visual polish):**
For .phase, .cycle, or other prominent text, consider adding:
```css
letter-spacing: clamp(0px, 0.2vw, 0.08em);
```

If letter-spacing already exists with fixed values, replace with clamp() formula.

**Rationale:** As font sizes scale with clamp(), line-height and letter-spacing should also scale proportionally. This maintains visual hierarchy and readability across all viewport sizes. Fixed line-height on scaled fonts can lead to cramped or too-loose spacing.

Verify after implementation:
- Body text on 320px phone: line-height should feel natural (not too tight)
- Body text on 4K desktop: line-height should feel natural (not too loose)
- Similar check for letter-spacing on headings

Do not worry about pixel-perfect consistency across all devices; clamp() is designed for smooth variation.
