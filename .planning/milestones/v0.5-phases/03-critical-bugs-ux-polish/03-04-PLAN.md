<objective>
Implement error/invalid input feedback for duration input validation using CSS-based red flash animation. When a user attempts to set duration outside valid 1-300s range, provide immediate visual feedback that the input was corrected.

**Purpose:** Silent value clamping (especially via HTML manipulation) can confuse users who expect 500s but get 300s. Red flash animation provides clear feedback that validation caught invalid input and corrected it, preventing user confusion about actual phase duration.

**Output:** 
- inputErrorFlash CSS @keyframes animation (red background flash)
- .durInput.error CSS class with border highlight
- Change handler integration to trigger error animation on clamped values
- 400ms animation duration for noticeable but non-disruptive feedback
</objective>

**Location 2 — Update duration change handler (lines ~1430-1444):**

Find the existing change listener `durationsEl.addEventListener("change", e => { ... })` and enhance it to show error animation when value is clamped:

```javascript
durationsEl.addEventListener("change", e => {
  const input = e.target.closest(".durInput");
  if (!input || isRunning) return;
  const idx = parseInt(input.dataset.index, 10);
  const raw = parseInt(input.value, 10);
  
  // Handle non-numeric input
  if (!Number.isFinite(raw)) {
    input.value = activePhases[idx].durationSec;
    input.classList.add("error");
    setTimeout(() => input.classList.remove("error"), 400);
    return;
  }
  
  // Clamp to DURATION_RANGE and show error if value changed
  const val = Math.min(DURATION_RANGE.max, Math.max(DURATION_RANGE.min, raw));
  if (val !== raw) {
    input.classList.add("error");
    setTimeout(() => input.classList.remove("error"), 400);
  }
  
  // Update input, state, and persist
  input.value = val;
  activePhases[idx] = { ...activePhases[idx], durationSec: val };
  reset();
  saveSettings();
});
```

**Why this approach:**
- Red background flash (rgba(229,115,115,0.2) = light red) is visible in both light and dark themes
- Border color change (#e57373 = red) provides additional emphasis
- 400ms animation is fast enough to feel immediate, slow enough to be noticeable
- Error class is removed after animation completes, preventing persistent styling
- No JavaScript performance overhead; 400ms timeout is acceptable for input validation
