      - Enforce cap: if (existing.length > maxHistoryEntries) existing.length = maxHistoryEntries — per D-14

      - Reset pagination: sessionStorage.setItem("historyPage", "1")

        - If dupCount > 0: `"Imported ${importedCount} sessions (${dupCount} duplicates skipped)"`
        - Otherwise: `"Imported ${importedCount} sessions"`
      - Call showImportFeedback(message)
      - Catch any error (including QuotaExceededError): call showImportFeedback("Import failed: invalid file format")

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system → app | User-selected .json import file crosses here; content is untrusted |
| app → localStorage | Merged/re-sorted array written back; must stay within quota |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Tampering | importJson — JSON.parse of user file | medium | mitigate | Array.isArray guard + per-entry schema validation (date string, durationMs finite, cycles finite); invalid entries skipped, not written |
| T-05-02 | Denial of Service | importJson — very large JSON file | low | accept | FileReader is browser-native; file size limited by OS file picker memory; localStorage write bounded by maxHistoryEntries cap after merge |
| T-05-03 | Information Disclosure | exportJson / exportCsv — file download | low | accept | User initiates download of their own data; no server involved; offline-only app; data never leaves device except to local filesystem |
| T-05-04 | Denial of Service | importJson — QuotaExceededError on localStorage.setItem | medium | mitigate | Wrap localStorage.setItem in try/catch; QuotaExceededError caught and shown as "Import failed: invalid file format" via showImportFeedback |
| T-05-05 | Tampering | clearHistBtn — accidental data deletion | high | mitigate | Native <dialog> confirmation with destructive-action language ("Delete all sessions"); ESC and outside-click dismiss without deleting; per D-15 |
</threat_model>
